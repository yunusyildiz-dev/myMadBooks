# Learning Principles & Milestones

## The Learning Cycle

Apply this cycle for **every chapter**:

1. **Read** — Read the chapter carefully, don't move on until you understand
2. **Write** — Type out the examples yourself (do NOT copy-paste!)
3. **Break** — Intentionally cause errors, read what the compiler says
4. **Extend** — Modify the examples, try new things
5. **Build** — Build something small with what you've learned

## Milestones

| Milestone | When | Checkpoint |
|-----------|------|------------|
| Ownership mastered | Week 4 | I can predict compiler errors before they happen |
| Traits + Generics fluency | Week 11 | I can write my own traits from scratch |
| minigrep complete | Week 13 | I have a working CLI tool |
| Concurrent program written | Week 21 | I use threads and async with confidence |
| Web server up and running | Week 28 | I built HTTP from TCP by hand |
| Backend API ready | Week 33 | Axum + SQLx + Auth working in production |
| Rust Developer | Week 35+ | I write production-quality Rust code |
